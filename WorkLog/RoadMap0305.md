# 多渠道AI智能体 - 核心模块功能与技术路线

## 1. 渠道接入模块 (Channel Connector)

**职责**: 作为统一网关，处理与各个前端渠道（飞书、钉钉、Web UI 等）的交互与协议转换。

**核心功能**:
- **多平台消息接收与发送**: 统一接收 Webhook/WebSocket 消息，抹平平台差异，将各种消息格式转换为内部标准格式。
- **权限与文件管理**: 
  - 识别用户身份与角色（如普通用户、群管理员）。
  - **核心链路**: 专门拦截并处理**群管理员上传的文件**（如PDF、Word、TXT等），将其作为知识库的唯一增量来源。
- **安全与校验**: 处理各平台的鉴权、加解密及 URL 验证。

**技术实现**:
```python
# 渠道接入与权限分发伪代码
@app.post("/webhook/channel")
async def handle_event(request: Request):
    event = parse_request(request)
    
    # 针对管理员上传文件的特殊处理
    if event.is_file_upload and event.user.role == "ADMIN":
        await trigger_knowledge_ingestion(event.file)
        return {"msg": "知识库更新成功"}
        
    # 普通消息交给路由器
    await dialog_router.process(event.message)
```

## 2. 对话路由器 (Dialog Router)

**职责**: 智能大脑的“交通调度员”，负责意图识别并控制对话流向。

**核心功能**:
- 意图识别与分类：精准区分用户的提问属于 **产品介绍 / 技术排障 / 商务咨询 / 闲聊** 等具体场景。
- 严格的流转控制逻辑。

**流转决策树**:
```text
用户消息 -> 意图分析分类 -> {
  [闲聊]: 
    -> 直接跳过检索，进入【AI对话引擎】进行日常互动。
    
  [产品介绍 / 技术排障 / 商务咨询]: 
    -> 必须首先路由至【知识库检索模块】进行搜索。
}
```

## 3. 知识库检索模块 (Knowledge Retriever)

**职责**: 智能体的“专属大脑外部记忆”，负责从管理员上传的权威文件中寻找答案依据。

**核心功能**:
- **知识库构建**: 仅接收【渠道接入模块】传来的、由**群内相关管理员上传的文件**，进行文本分块与向量化，以此丰富和更新专属知识库。不采纳未经授权的数据。
- **精准检索**: 根据路由分发过来的专业问题，在向量数据库中进行语义检索。
- **结果流转决策**:
  - **有搜索结果**: 提取相关文档片段，传递给下一个环节【AI对话引擎】进行提炼和回复。
  - **未搜索到结果**: 携带“未命中”状态，进入下一个环节【AI对话引擎】，请求大模型通过通用知识库搜索答案或生成引导兜底话术。

## 4. AI对话引擎 (AI Dialog Engine)

**职责**: 系统的最终表达中枢，负责理解上下文、消化检索资料并生成拟人化的自然语言回复。

**核心功能**:
- **结合语境生成回答**: 
  - 针对**已在知识库中搜索到**的内容：基于检索到的文档片段，提炼总结出精准、易懂的回答，坚决不脱离文档“幻觉”编造。
  - 针对**知识库未搜索到**的内容：利用大模型自身的知识深度来尝试搜索答案、解答问题，或坦诚告知超出现有资料范围并建议联系人工。
  - 针对**闲聊**：结合赋予的人设进行得体的日常对话。
- **会话管理**: 维护多轮对话的 Memory，保证聊天的连贯性。

**Prompt工程示例**:
```text
系统提示词(System Prompt):
- 你是专业的智能助手。
- 请优先基于下方【检索到的企业知识】来回答问题。
- 如果【检索到的企业知识】为空，你可以根据你的通用知识进行解答，但请注意回答的严谨性。

历史上下文: {History_Context}
检索到的企业知识: {Retrieved_Knowledge} (可能为空)
用户问题: {User_Query}
```

## 5. 产品知识管理 (Product Knowledge - 高频记忆区)

**职责**: 作为一个高频词条与常见问题的“记忆缓存区”，专门用于应对重复性极高的相似提问。

**核心功能与定位**:
- **高频问题记忆 (FAQ Cache)**: 存储历史沉淀的高频问答（如“如何收费”、“支持哪些系统”）。
- **专业词条字典**: 存储业务专有词汇的固定解释。
- **作用**: 当用户的问题与该记忆区中的词条或FAQ高度相似时，可以直接快速响应。这避免了每次类似问题都要去大文档（PDF等）中进行繁重的向量检索和长文本推理，极大提升了系统响应速度并降低了算力成本。

**数据格式**:
```json
{
  "entry_type": "hot_faq",
  "keywords": ["收费", "价格", "多少钱"],
  "standard_question": "你们的产品是如何收费的？",
  "quick_answer": "我们提供标准版和企业版，标准版免费，企业版根据用户数阶梯收费。详细报价单请见..."
}
```
