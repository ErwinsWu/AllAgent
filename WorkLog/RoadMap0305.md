# 多渠道AI智能体 - 技术演进路线图

本项目采取敏捷开发与分阶段迭代的策略，从核心逻辑验证逐步演进到多渠道、高性能的企业级生产环境。

## Phase 1: 核心框架搭建与单渠道MVP (第1-2周)

**目标**: 跑通核心链路，验证对话与基础知识库的可行性。

**关键里程碑**:
- [ ] **基础设施选型**: 搭建基于 `FastAPI` 的后端基础工程。
- [ ] **单渠道接入**: 优先完成单一高频渠道（例如飞书）的 Webhook 接入与消息加解密。
- [ ] **基础对话路由**: 实现简单的意图分类（基于规则或轻量级 Prompt分类）。
- [ ] **LLM 最小闭环**: 接入大语言模型 API（如 OpenAI 或 Claude），完成纯文本对话功能的调试。
- [ ] **记忆区打样**: 以硬编码或静态 JSON 文件的形式，实现【产品知识管理 (FAQ缓存)】的快速响应。

## Phase 2: 知识库引擎构建与权限闭环 (第3-4周)

**目标**: 引入外部真实知识，实现基于权威文档的 RAG (检索增强生成) 机制。

**关键里程碑**:
- [ ] **向量数据库部署**: 引入并部署 ChromaDB 或 Milvus 作为核心向量存储。
- [ ] **权限与文件处理**: 
  - 实现管理员身份校验。
  - 开发文档解析服务（支持 PDF、Word、TXT 等常见格式的文本提取与分块 Chunking）。
- [ ] **语义检索闭环**:
  - 接入 Embedding 模型将文本块向量化。
  - 实现 Top-K 语义召回并注入到 LLM 的 Prompt 上下文中。
- [ ] **防幻觉机制**: 在 Prompt 和路由层增加兜底逻辑，确保无搜索结果时引导至人工。

## Phase 3: 渠道横向拓展与交互升级 (第5-6周)

**目标**: 将单点能力泛化，支持多平台并提供更丰富的呈现形式。

**关键里程碑**:
- [ ] **协议适配器层开发**: 彻底解耦业务逻辑与平台协议，新增钉钉、企业微信等渠道支持。
- [ ] **自建 Web 界面**: 开发基于 Vue/React 的独立前端界面，通过 WebSocket 或 SSE 实现流式打字机输出效果。
- [ ] **富文本与卡片消息**: 升级【渠道接入模块】，支持将 AI 生成的 Markdown 结果转换为各平台原生的富文本卡片（如飞书消息卡片），提升阅读体验。
- [ ] **多轮会话优化**: 引入 Redis 缓存机制，实现高效的跨轮次上下文 Memory 管理。

## Phase 4: 模型私有化部署与推理优化 (第7-8周及长期)

**目标**: 降低 API 调用成本，保障数据绝对安全，提升系统极限并发与响应速度。

**关键里程碑**:
- [ ] **本地模型微调与部署**: 探索使用开源模型（如 Qwen、Llama 等）结合企业脱敏数据进行微调，并进行本地化部署。
- [ ] **计算图与算子优化**: 针对选定的底层模型结构，探索将其导出为 ONNX 格式，打通跨平台推理链路。
- [ ] **推理加速引擎**: 引入 TensorRT 等高性能推理框架替代原生 Python 推理。
- [ ] **模型量化 (Quantization)**: 实施 INT8/INT4 模型量化方案，在保证回复质量（Perplexity）的前提下，大幅降低显存占用并提升端到端的吞吐量 (Throughput)。
- [ ] **系统全链路监控**: 接入 Prometheus + Grafana，监控各渠道消息延迟、向量检索耗时、模型推理时延及 Token 消耗。
