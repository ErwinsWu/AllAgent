### 🎯 算法研发任务书：客户全景图谱大模型信息抽取

**To: 算法负责人**
**From: PM**

#### 一、 核心目标与定位
本阶段你的核心任务**不是**去做复杂的 RAG 文档切分或底层模型微调。
你的唯一目标是：**编写并调优一段极其稳定的 Prompt（提示词），让大模型在阅读一段非结构化的客户历史聊天记录/CRM 备注后，能够 100% 稳定地输出我们规定好的 JSON 格式数据。**


#### 二、 标准 JSON Schema（前后端对接合同）
大模型的输出必须严格符合以下结构。前期测试时，如果真实文档数据缺失，允许让大模型根据上下文“合理伪造”文件名（Mock 数据），以跑通整体数据流为第一优先级。

```json
{
  "customer_profile": {
    "customer_name": "客户名称或代号",
    "value_tier": "高价值/中等价值/低价值", 
    "risk_level": "安全/待激活/高危流失", 
    "persona_tags": ["标签1", "标签2", "标签3"] 
  },
  "historical_needs": [
    {
      "need_id": "N001",
      "topic": "历史核心诉求简述（如：需要私有化部署方案）",
      "status": "已满足/推进中/未满足",
      "associated_docs": [
        {"doc_name": "私有化部署技术白皮书.pdf", "doc_type": "PDF"}
      ]
    }
  ],
  "current_blockers": [
    {
      "blocker_id": "B001",
      "topic": "当前阻碍成单/续费的核心卡点（如：今年预算腰斩）",
      "severity": "高/中/低",
      "details_summary": "提取聊天中的具体痛点描述，限50字以内"
    }
  ],
  "retention_strategy": {
    "strategy_type": "降价挽留/价值自证/情感关怀/干预实施",
    "next_best_action": "下一步业务员应该做的具体动作（如：发送竞品ROI对比表并申请9折权限）",
    "suggested_materials": [
      {"doc_name": "2026版竞品对比分析表.xlsx", "doc_type": "Excel"}
    ],
    "auto_reply_script": "根据客户语境，生成一段业务员可以直接复制发送的专业话术"
  }
}
```

#### 三、 核心节点业务逻辑说明（调优 Prompt 的依据）

你需要让大模型理解这些字段背后的**业务意义**，才能抽得准：

1.  **`customer_profile` (静态画像区)：** * **业务目的：** 让接手的新业务员0成本了解客户底色。
    * **抽取重点：** `risk_level`（风险等级）极其关键。必须在 Prompt 中定义清楚什么是“高危流失”（例如：提到竞品、不回微信超过3次、明确表示太贵）。
2.  **`historical_needs` (过往诉求区)：** * **业务目的：** 盘点公司对这个客户的“履约情况”。如果一堆“未满足”，业务员就知道这客户带着怨气。
    * **抽取重点：** 这里的 `associated_docs`（关联文档）是为后续接入 RAG 留的接口。现在大模型可以提取上下文中提到的文件名，或者根据常识推断一个。
3.  **`current_blockers` (当前卡点区 - 重中之重)：** * **业务目的：** 揪出客户现在不付钱的根本原因。
    * **抽取重点：** 必须准确归纳。前端图谱上会把 `severity` 为“高”的节点渲染成红色警告，这是业务员视觉的第一落点。
4.  **`retention_strategy` (维系策略区 - 我们的核心卖点)：** * **业务目的：** AI 军师的直接体现。把图谱分析转化为具体的“动作（Action）”。
    * **抽取重点：** `auto_reply_script` 必须是**“说人话”**的销售口吻，严禁机器味太重。

#### 四、 交付标准与验收

1.  **接口稳定性：** 提供 5 段不同长度、不同语气的真实聊天记录文本等测试用例。你的接口必须能成功解析，且输出的 JSON 格式不能崩溃（如缺少括号、字段名拼写错误等）。
2.  **格式强制性：** 建议使用大模型的 `JSON Mode`（如有），或者在 Prompt 尾部强调：“严格输出纯 JSON，不包含任何 Markdown 标记（如 ```json），不包含任何解释性废话。”
