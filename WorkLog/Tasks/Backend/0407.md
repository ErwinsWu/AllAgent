### 📅 4月第2周开发冲刺计划：视觉与证据链专项 (4.7 - 4.14)

---
#### 🧠 后端开发任务：证据链提取与文件解析管道 (重量级)

* **Task 1：Prompt 升级：增加“推理与溯源”字段 (替换高并发任务)**
    * **开发内容：** 修改后端提取 JSON 的 System Prompt。要求大模型在输出每个标签（如：高危流失）时，必须同时抓取原对话中对应的**原文证据**。
    * **示例 JSON 变化：** `{"tag": "价格敏感", "evidence": "客户在 10:15 提到：‘易云管家报价比你们便宜一半’"}`。
    * **交付物：** 更新后的 API 接口，返回的 JSON 包含 `evidence` 字段
* **Task 2：“每日投喂口”文件解析引擎 (核心攻坚)**
    * **开发内容：** 编写 `/upload-knowledge` 接口，集成 `PyMuPDF`。
    * **交付物：** 实现 PDF 和 TXT 文件的上传与纯文本提取，并在后端控制台打印解析结果。
* **Task 3：全栈支援 - 前后端链路缝合**
    * **开发内容：** 在前端页面实现一个极其基础的上传文件按钮，并将后端新提取的 `evidence` 字段绑定到前端写的“抽屉”组件中。
    * **交付物：** 演示链路通畅：点击图谱节点 -> 抽屉打开 -> 看到后端抽取的“原文证据”。（如果无法联调，可以延后，但保证后端返回结果一致）
