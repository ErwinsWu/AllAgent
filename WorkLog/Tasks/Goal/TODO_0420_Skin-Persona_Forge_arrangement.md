# 🚀 客户留存 Agent V2.0 产品架构规划书：全场景电子皮肤定制塑形舱

## 🎯 一、 产品定位与演进愿景
V1.0 版本的“留存图谱”解决了“看清客户痛点”的问题；
V2.0 版本将打造**【电子皮肤人格塑形舱 (Skin-Persona Forge)】**。
打通线上企微与线下会议全场景，并实现历史数据的无缝继承，将客户模糊的需求直接转化为可供技术人员参考的开发需求，实现“需求即代码”的零距离交付

## ⚙️ 二、 全局双模式切换与数据联动架构
系统前端支持业务员在两大模式间丝滑切换，且底层数据完全贯通：
* **Mode 1：留存维系模式（基础盘）：** ECharts 蜘蛛网图谱，用于商机跟进、痛点排查。
* **Mode 2：定制塑形模式（新增核心）：** 当客户确认导入电子皮肤方案时切入。**进入此模式时，系统会自动继承 Mode 1 中沉淀的客户画像数据，实现“热启动”。**

---

## 🧬 三、 人格塑形舱 (Skin-Persona Forge) 四步核心业务流

### 🎬 第一阶段：双径信息注入与战前筹备 (Initiation & Briefing)
在留存维系模式页面上有一个按钮，启动塑形舱，业务员点击 `[启动塑形舱]`，界面切换为科技感“四环槽位”UI（应用场景、交互性格、触觉阈值、禁忌指令）。

**【数据注入引擎（热启动）】**
Agent 在开启瞬间，提供两种获取客户基础信息的路径：
* **路径 1（自动继承提取）：** 业务员选择“从当前对话生成”。Agent 后端自动抓取该客户在【留存图谱】中已有的画像标签（如：*高净值、医疗器械行业、曾抱怨过防水问题*）以及前序聊天记录，自动填充到塑形舱的基础底表中。
* **路径 2（人工覆盖/新建）：** 面对全新客户，业务员可以手动输入核心背景（如：*新拓客户，扫地机器人行业*）。

信息注入后，Agent 提供两套作战准备方案：

**🔴 场景 A：准备【线下会议/拜访】**
基于注入的客户信息，Agent 有一个按钮自动生成一份**《线下深度需求采集问诊单 (Sequential Checklist)》**。
* **逻辑特征：** 严格按照线下会议的时间轴和人脑逻辑递进顺序生成的 SOP，数量会较多：
    1.  **破冰与定调：** “张总，这款复健机器人的核心人群是术后老人还是工伤青年？”（推导 `应用场景`）
    2.  **核心触觉探测：** “需要防汗液误触吗？”（推导 `触觉阈值`）
    3.  **情感反馈界定：** “如果患者锤击机器人，希望它的语音是警告还是安抚？”（推导 `交互性格`）
    4.  **安全底线与禁忌：** “医疗合规层面有哪些绝对不能出现的词汇？”（推导 `禁忌指令`）

**🔵 场景 B：开启【线上企微问询】**
业务员点击 `[开启线上向导]`，Agent 根据注入的客户画像，在侧边栏生成针对性的首发破冰话术，准备进入动态对话监听状态。

### 🎬 第二阶段：双轨数据捕获与动态重塑 (Live & Offline Extraction)

**🔴 轨道 A：线下会议纪要极速解析**
* **输入：** 业务员将线下开会产生的长篇会议纪要或飞书转写文本，直接丢进 `[会议纪要解析入舱]` 上传框。
* **输出：** 大模型穿透废话，瞬间提炼核心需求，UI 界面的四个光环瞬间被准确的标签填满（如：`[防汗液]`, `[温柔女声]`）。

**🔵 轨道 B：线上动态追问引擎（Dynamic Streaming Inquiry）**
* **输入：** 业务员与客户在企微一问一答，Agent 实时读取聊天流。
* **核心逻辑（动态状态机）：** 具备“听音辨位”的追问能力。
    * *Agent 依据历史数据推送基础问题：* “您好，咱们这款医疗设备的受众群体是？”
    * *客户回复：* “主要是5岁患儿，比较好动。” -> UI 瞬间亮起标签 `[5岁儿童]`。
    * *Agent 动态生成下一步提问：* 检测到“好动儿童”，结合之前继承的“医疗器械”属性，**立刻调整顺位并生成针对性追问**发送给业务员：“*建议追问：考虑到好动患儿可能会拉扯设备，您需要我们将电子皮肤的『防撕裂报警阈值』拉到最高吗？*”
    * *效果：* 业务员像拥有了一个顶级产品经理在旁辅助，精准收敛客户需求。

### 🎬 第三阶段：微调干预与共识确认 (Fine-tuning & Consensus)
* **全息预览板：** 侧边栏展开所有被抽取的标签。
* **人工校验：** 业务员点击 `[x]` 删除冗余标签，或点击 `[+]` 补充要求。
* **一键共识：** Agent 自动根据当前定稿的标签，生成一段高度专业的需求确认话术，业务员一键发送给客户进行商务确认。

### 🎬 第四阶段：一键炼丹与代码烧录 (Synthesis & Deployment)
* **动作：** 客户确认后，业务员点击 `[编译皮肤系统prompt]`。
* **产出物交付：** Agent 在界面输出两组核心数据：
    1.  **灵魂代码 (System Prompt)：** 限定机器人性格、语气和禁忌的大模型系统指令。
    2.  **躯壳代码 (Hardware JSON)：** 暂定。
