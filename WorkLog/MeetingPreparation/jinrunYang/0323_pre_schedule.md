### 客户留存 Agent：4 个月极速落地执行手册 

#### 🛡️ 破局共识（今后规矩）
1.  **停止底层造轮子：** 直接用现成 API
2.  **砍掉多模态：** 第一版 Demo 绝对不碰图片、表格解析。所有客户资料、产品文档全部手动转成纯文本 `.txt`。
3.  **异步开发，假数据先行：** 软件同学不能等算法同学的接口。提供一份标准的 JSON 假数据，软件同学必须看着假数据把页面画出来。

---

#### 📍 第 1 个月（4月）：跑通骨架，眼见为实 (Mock 驱动期)
**本月目标：不要真实的 AI，只要一个“看起来像那么回事”的企微/飞书侧边栏和大屏。**

* **👨‍💻 软件同学（主攻期）：**
    * **具体动作：** 注册企微/飞书开发者，创建一个侧边栏应用。写一个包含两个按钮（“生成话术”、“查看全景图”）的侧边栏 H5 页面。
    * **痛点解法：** 会卡在官方鉴权（OAuth）。哪怕界面丑成毛坯房，只要能弹出一个框显示 `当前客户ID: 12345`，这个月他的首要任务就算圆满完成。
    * **交付物：** 企微内可点击的侧边栏；点击“查看全景图”能弹出一个网页，里面是用 ECharts 配合**假 JSON 数据**画出来的一张静态蜘蛛网图。
* **🧠 算法同学（减负期）：**
    * **具体动作：** 停止知识库调参。用 Python (FastAPI) 写一个最简单的 Web 后端框架。写一个接口（API），前端一调用，后端就返回那串固定的“假 JSON 数据”。
    * **痛点解法：** 从繁重的理论学习中抽身，先建立“前端调后端，后端给数据”的工程思维。
    * **交付物：** 一个能跑通 GET/POST 请求的本地接口。
* **🎯 PM的任务（查验与协助）：**
    * **动作：** 产出那份至关重要的**《V1版图谱 JSON Schema》**（比如规定必须有：客户标签、历史痛点、跟进建议三个字段）。编造 1 份极其逼真的“客户 A 的 JSON 数据”。
    * **查验点：** 亲自在电脑上点开企微，确认侧边栏能打开，点击大窗能看到那张给的假数据画出来的蜘蛛网。

#### 📍 第 2 个月（5月）：注入灵魂，文本抽结构 (Prompt 攻坚期)
**本月目标：用真实的脱敏聊天记录，自动生成那串 JSON 数据。**

* **🧠 算法同学（主攻期）：**
    * **具体动作：** 死磕 Prompt Engineering（提示词工程）。将一段几百字的销售聊天记录扔给大模型 API，通过不断调整提示词，逼迫大模型稳定地输出前端需要的 JSON 格式。
    * **痛点解法：** 大模型输出格式容易崩。学会使用模型厂商提供的 `JSON Mode`（强制 JSON 输出功能），或者用正则表达式做一层保底清洗。
    * **交付物：** 一个真正的 API 接口——传入一堆聊天文本，吐出格式完美的结构化 JSON。
* **👨‍💻 软件同学（配合期）：**
    * **具体动作：** 将侧边栏里写死的假数据替换为调用算法同学的真实 API。实现大窗蜘蛛网图的**可点击交互**（例如，点击“嫌弃价格”节点，右侧滑出一个小抽屉，显示对应的原始聊天记录片段）。
    * **交付物：** 侧边栏大屏实现动态渲染和节点点击交互。
* **🎯 PM的任务（查验与协助）：**
    * **动作：** 化身“数据标注员”。准备 20 份脱敏的真实聊天记录。
    * **查验点：** 拿这 20 份记录去测算法同学的接口，提取出来的“客户痛点”如果不准，把 Bad Case 整理出来，打回去让他改 Prompt。不要让他闭门造车，PM来定义“提取准不准”。

#### 📍 第 3 个月（6月）：外挂知识，RAG 极简接入 (业务闭环期)
**本月目标：让 Agent 根据痛点给出建议，并挂载极简版知识库。**

* **🧠 算法同学（主攻期）：**
    * **具体动作：** 将之前实验的 RAG 知识库做**极度精简**。不要各种复杂文件，只切分 10 份纯文本的销售话术/产品 QA (`.txt`)。在生成 JSON 的同时，顺便检索出最相关的 1 条 QA，附在 JSON 的 `推荐话术` 字段里。
    * **痛点解法：** 严禁纠结召回率。只要能根据关键词（如“打折”）匹配到一段 txt 文本即可，先用最基础的向量对比跑通。
    * **交付物：** API 输出的 JSON 中，新增了精准的“下一步动作建议”和“参考话术”。
* **👨‍💻 软件同学（配合期）：**
    * **具体动作：** 在侧边栏增加“一键回填”功能。调用企微的 `sendChatMessage` 接口，点击侧边栏上 AI 给出的建议话术，直接把文字填入左侧聊天输入框。
    * **交付物：** 业务员可以一键把 AI 建议发给客户，完成业务闭环。
* **🎯 PM的任务（查验与协助）：**
    * **动作：** 精心挑选并纯文本化那 10 份最核心的销售话术 QA，喂给算法同学。
    * **查验点：** 扮演业务员，点一下侧边栏按钮，看生成的文字能不能丝滑地出现在输入框里。

#### 📍 第 4 个月（7月）：打磨剧本，准备大考 (Demo 冲刺期)
**本月目标：绝对稳定，不允许任何新功能加入，为汇报演示做保底。**

* **👨‍💻 软件同学 & 🧠 算法同学（联合防守）：**
    * **具体动作：** 修复前端 UI 错位、加载过慢（加 Loading 动画）等体验问题。
    * **强制动作（保命符）：** 两人必须配合，在系统里写一个隐藏的**“演示模式开关”**。一旦开启，系统切断所有网络大模型 API 请求，完全读取本地准备好的完美静态数据。
    * **交付物：** 一个即使在断网或大模型服务器宕机的情况下，依然能完美演示完所有点击流程的系统。
* **🎯 PM的任务（主攻汇报）：**
    * **动作：** 撰写《汇报演示剧本》。具体到“我先点击哪个人，再说哪句话，然后鼠标滑向哪个节点”。
    * **查验点：** 用这套系统连续走 10 遍剧本，确保每一个环节肌肉记忆般顺滑。
