### 开发碰头会：7月Demo冲刺方案周会（修正版）

#### 一、 目标对齐：产品形态升级 (Sidebar + Big Screen)
我们要实现一个“双屏联动”的交互体验：
* **侧边栏（随身 Agent）：** 依然挂载在企微/飞书聊天界面的右侧。主要功能是**自动提醒、AI建议生成、以及与当前客户相关的快速问答**。
* **全景大窗（客户 360° 图谱）：** 侧边栏里有一个核心按钮【查看客户全景图】。点击后，通过企微/飞书的 API 直接弹出一个大窗口（或跳转外部浏览器）。
* **图谱深度交互（新亮点）：** 大窗里的“蜘蛛网”不再是静态图片。每一个节点（如“竞品比价”、“预算卡点”）都可以点击。点击后，右侧滑出一个面板，直接展示相关的历史聊天记录截图、产品说明书 PDF 链接、或者之前整理的 RAG 知识库片段。

#### 二、 团队分工：谁来做什么？（核心策略：让专业的人做专业的事）

为了保证业余时间的高效产出，必须明确各自的“主战场”。

**1. 算法软件同学 —— 专注“最强大脑”与“数据结构”（核心壁垒）**
*定位：不用管界面好不好看，不用管飞书API怎么接，全身心投入让AI变聪明。*
* **信息抽取转 JSON：** 编写复杂的 Prompt，让大模型能精准阅读脱敏的聊天记录，并吐出完美的、包含节点层级关系的 JSON 数据结构。
* **RAG 节点挂载逻辑：** 结合之前做的 RAG 测试，思考如何让大模型在提取“痛点”节点时，顺便把相关的本地文档或知识库文本 ID 关联到这个 JSON 节点里。
* **智能回复大脑：** 负责侧边栏那个“预测下一步动作”的 AI 逻辑。

**2. 软件执行同学 —— 专注“业务落地”与“用户体验”（产品门面）**
*定位：你是这个Demo最终效果的呈现者，负责打通企微/飞书的生态，把算法同学的“脑力”变成可以点的“产品”。*
* **企微/飞书 API 对接：** 搞定开发者后台，写代码获取当前客户 ID，实现侧边栏的挂载，以及“点击按钮弹出大窗口”的官方接口调用。
* **前端网页生成与绘图：** 利用 AI 辅助写代码，用 Vue/React 写侧边栏网页和全景大窗网页。引入 ECharts 或 AntV G6 库，把算法同学传过来的 JSON 画成酷炫的、可点击的蜘蛛网图。
* **前后端联调路由：** 搭建简单的后台（如 Python FastAPI 或 Node.js），把前端的点击动作传递给算法侧，再把结果返回给前端展示。

#### 三、 接下来两周的“晚间冲刺”具体任务排期

* **第 1 周：跑通骨架（不用真AI，用假数据）**
    * **软件同学：** 跑通企微/飞书侧边栏“Hello World”。写一个静态的大窗网页，用假的 JSON 数据画出一个带有两三个节点的 ECharts 图，实现“点击节点弹出详情文字”的交互。
    * **算法同学：** 抛开代码，在大模型网页端（如 Kimi 或 ChatGPT）死磕 Prompt。跑通 `输入一段聊天 -> 稳定输出预设 JSON 格式` 这一关。
    * **你（项目负责人）：** 准备 1 份完美的虚构客户案例（包含完整的聊天记录、对应的 JSON 结构、节点背后挂载的假文档文件）。

* **第 2 周：神经互联（前后端对接）**
    * **双方协作：** 软件同学的侧边栏发送请求 -> 后台接收 -> 调用算法同学写好的大模型接口 -> 生成真实 JSON -> 传回前端大窗渲染图谱。
    * **你（项目负责人）：** 开始梳理业务话术，构思 Demo 演示当天的剧本，测试前端交互是否顺滑。

#### 四、 绝对优先级


* **任务 A：定义“蜘蛛网”的数据结构 (JSON Schema)。** 定义好：图谱里到底有哪几个核心维度（比如：客户画像、历史接触、痛点卡点、干预建议）。
* **任务 B：准备“弹药”和“粮草”。** 收集并脱敏 3-5 份真实的客户沟通记录。准备几个你们产品线的 PDF 文档或常见问答 txt，用来给他们做 RAG 挂载节点的测试。
* **任务 C：扫清行政与账号障碍。** 确保飞书/企微的管理员权限、大模型 API 的调用额度（Key）都已经准备就绪，不要让开发同学因为没有账号而停工。
